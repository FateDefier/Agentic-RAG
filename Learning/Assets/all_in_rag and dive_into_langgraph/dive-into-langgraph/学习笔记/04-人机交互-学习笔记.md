## 要点整理

### 一、人机交互（HITL）概述

- **定义**：Agent 主动中断执行，向人类索要执行权限或额外信息，获得反馈后继续执行
- **实现方式**：通过 `HumanInTheLoopMiddleware`（HITL）中间件实现
- **工作流程**：Agent 执行 → 触发中断 → 保存状态到 Checkpoint → 等待人类回复 → 恢复状态 → 继续执行

### 二、HITL 配置详解

```python
HumanInTheLoopMiddleware(
    interrupt_on={
        "tool_a": False,        # 无需审批
        "tool_b": True,         # 需要审批（三种审批类型）
        "tool_c": {"allowed_decisions": ["approve", "reject"]}  # 仅允许通过和拒绝
    },
    description_prefix="Tool execution pending approval",
)
```

| 配置值 | 效果 |
|--------|------|
| `False` | 不触发审批，工具自动执行 |
| `True` | 触发审批，支持 approve / edit / reject 三种决策 |
| `{"allowed_decisions": [...]}` | 触发审批，但只允许指定的决策类型 |

### 三、检查点（Checkpoint）方案

| 方案 | 存储方式 | 生产适用 | 特点 |
|------|----------|----------|------|
| `InMemorySaver` | 内存 | ❌ | 速度快，程序重启后丢失 |
| `SqliteSaver` | SQLite 文件 | ✅ | 轻量级，适合单机部署 |
| `PostgresSaver` | PostgreSQL | ✅ | 支持分布式，数据可靠 |
| `MongoDBSaver` | MongoDB | ✅ | 文档型，灵活扩展 |
| `RedisSaver` | Redis | ✅ | 高速缓存，适合高频读写 |

### 四、恢复执行

```python
# 审批通过
Command(resume={"decisions": [{"type": "approve"}]})

# 修改参数
Command(resume={"decisions": [{"type": "edit", "args": {...}}]})

# 拒绝
Command(resume={"decisions": [{"type": "reject"}]})
```

### 五、内置中间件类

| 中间件 | 功能 |
|--------|------|
| `SummarizationMiddleware` | 接近 token 上限时自动摘要 |
| `ModelCallLimitMiddleware` | 限制模型调用次数 |
| `ToolCallLimitMiddleware` | 限制工具调用次数 |
| `ModelFallbackMiddleware` | 主模型失败时自动回退到备用模型 |

## 参考答案汇总

### 基础题答案

1. **人机交互（HITL）**：指 Agent 在执行过程中主动中断，向人类请求审批或额外信息，待人类反馈后继续执行的机制。在 LangGraph 中通过 `HumanInTheLoopMiddleware` 中间件实现，核心依赖检查点（Checkpointer）保存和恢复 Agent 的执行状态。

2. **`interrupt_on` 三种配置方式：**
   - `False`：不触发中断，工具自动执行
   - `True`：触发中断，允许 approve（通过）、edit（修改参数）、reject（拒绝）三种审批类型
   - 字典配置：`{"allowed_decisions": ["approve", "reject"]}`，触发中断但限定审批类型

3. **检查点（Checkpoint）的作用**：在 Agent 被中断时保存当前的完整状态（包括消息历史、节点执行位置等），待人类反馈后可以精确恢复执行。生产环境支持 `SqliteSaver`、`PostgresSaver`、`MongoDBSaver`、`RedisSaver` 等持久化方案。

4. **恢复执行的方式**：通过 `agent.invoke(Command(resume=...), config=config)` 恢复。`resume` 支持三种操作：
   - `{"decisions": [{"type": "approve"}]}`：审批通过
   - `{"decisions": [{"type": "edit", "args": {...}}]}`：修改参数后通过
   - `{"decisions": [{"type": "reject"}]}`：拒绝执行

5. **`description_prefix` 参数**：在中断信息前添加自定义前缀描述，帮助人类理解当前需要审批的是什么操作。

6. **内置中间件类**：
   - `SummarizationMiddleware`：对话摘要
   - `ModelCallLimitMiddleware`：模型调用次数限制
   - `ToolCallLimitMiddleware`：工具调用限制
   - `ModelFallbackMiddleware`：模型回退
   - `HumanInTheLoopMiddleware`：人机交互
   - `DynamicPromptMiddleware`：动态提示词

### 进阶题参考答案

1. **分级审批方案设计：**
   ```python
   # 在工具函数中嵌入金额信息
   @tool
   def process_payment(amount: float, ...):
       """处理支付"""
       return ...
   
   # 创建三个不同的 Agent，分别配置不同的 HITL
   # 或在一个 Agent 中通过 middleware 动态判断
   # 更优雅的方案：使用自定义中间件检查金额
   @before_agent
   def approval_routing(state, runtime):
       # 解析工具调用参数中的金额
       # 根据金额大小动态设置中断行为
   ```

2. **检查点方案选型考量：**
   - `InMemorySaver`：开发测试用，不适合生产
   - `SqliteSaver`：小型项目，单机部署，简单可靠
   - `PostgresSaver`：企业级，需要事务支持和多机部署
   - `RedisSaver`：高频读写场景，但 Redis 本身不保证数据持久性（需配置 AOF/RDB）

3. **`edit` 审批的数据流转：** 用户通过 `Command(resume={"decisions": [{"type": "edit", "args": {"city": "Beijing"}}]})` 提交修改后的参数 → LangGraph 从 Checkpoint 恢复中断时的状态 → 用用户提供的 `args` 覆盖原始的 tool call 参数 → 工具使用修改后的参数执行 → 结果返回给 Agent。

### 编程题参考实现

```python
import os
import uuid
import sqlite3
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain.agents import create_agent
from langchain.agents.middleware import HumanInTheLoopMiddleware
from langchain.tools import tool
from langgraph.checkpoint.sqlite import SqliteSaver
from langgraph.types import Command

_ = load_dotenv()

llm = ChatOpenAI(
    model="qwen3-coder-plus",
    api_key=os.getenv("DASHSCOPE_API_KEY"),
    base_url=os.getenv("DASHSCOPE_BASE_URL"),
)

@tool
def run_python_code(code: str) -> str:
    """执行 Python 代码并返回结果"""
    try:
        exec_globals = {}
        exec(code, exec_globals)
        return "代码执行成功"
    except Exception as e:
        return f"执行错误: {str(e)}"

@tool
def read_file(path: str) -> str:
    """读取文件内容"""
    try:
        with open(path, 'r', encoding='utf-8') as f:
            return f.read()
    except Exception as e:
        return f"读取失败: {str(e)}"

@tool
def delete_file(path: str) -> str:
    """删除指定文件"""
    try:
        os.remove(path)
        return f"文件 {path} 已删除"
    except Exception as e:
        return f"删除失败: {str(e)}"

# 创建 SqliteSaver
if os.path.exists("hitl-demo.db"):
    os.remove("hitl-demo.db")
checkpointer = SqliteSaver(
    sqlite3.connect("hitl-demo.db", check_same_thread=False)
)

# 创建带 HITL 的 Agent
agent = create_agent(
    model=llm,
    tools=[run_python_code, read_file, delete_file],
    middleware=[
        HumanInTheLoopMiddleware(
            interrupt_on={
                "read_file": False,
                "run_python_code": True,
                "delete_file": {"allowed_decisions": ["approve", "reject"]},
            },
            description_prefix="请审批以下操作",
        ),
    ],
    checkpointer=checkpointer,
    system_prompt="You are a helpful assistant that can execute code and manage files.",
)

# 运行并模拟审批通过
config = {"configurable": {"thread_id": str(uuid.uuid4())}}
result = agent.invoke(
    {"messages": [{"role": "user", "content": "请执行 print('Hello World')"}]},
    config=config,
)
# 审批通过
result = agent.invoke(
    Command(resume={"decisions": [{"type": "approve"}]}),
    config=config,
)

# 模拟审批拒绝
config2 = {"configurable": {"thread_id": str(uuid.uuid4())}}
result = agent.invoke(
    {"messages": [{"role": "user", "content": "请执行 print('test')"}]},
    config=config2,
)
result = agent.invoke(
    Command(resume={"decisions": [{"type": "reject"}]}),
    config=config2,
)
```

### 大厂面试题参考解答

1. **【百度】必须引入 HITL 的场景：**
   - 涉及资金操作（支付、转账）
   - 涉及数据删除/修改（DELETE/UPDATE 操作）
   - 涉及敏感权限（访问用户隐私数据）
   - 决策边界模糊的高风险操作
   - 法规要求的合规审批流程

2. **【大厂通用】中断机制原理：** 当 Agent 执行到需要审批的工具调用时，HITL 中间件会拦截执行流程，将当前 State 保存到 Checkpointer 中，然后返回 `__interrupt__` 信号。Checkpoint 保存了完整的状态快照（消息列表、节点执行状态等），使得后续可以从精确的中断点恢复执行，而不是从头重试。

3. **【字节跳动】平衡用户体验和安全：**
   - 审批粒度策略：低风险操作自动放行，中风险操作简单确认，高风险操作详细审批
   - 避免过度审批：设置合理的审批阈值和范围
   - 提供「不再提醒」选项（适用于同一会话中的同类操作）
   - 支持批量审批和审批模板

4. **【阿里】时间旅行（Time Travel）与 HITL 的关系：** 时间旅行利用 Checkpoint 保存的每个执行步骤的状态快照，允许开发者回溯到任意历史状态点重新执行。HITL 在中断时保存的 Checkpoint 天然支持时间旅行——如果审批结果不满意，可以回溯到中断点之前的状态，修改输入后重新尝试。这是 Checkpoint 机制的延伸应用。
