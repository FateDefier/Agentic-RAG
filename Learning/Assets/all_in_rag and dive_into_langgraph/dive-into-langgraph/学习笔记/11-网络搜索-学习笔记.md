## 要点整理

### 网络搜索的三项核心任务
- **网页爬取**：自动化抓取互联网上的网页内容
- **网页索引**：对爬取的内容建立结构化索引（几乎只有大公司能做）
- **网页检索**：根据用户查询返回相关结果

> SearXNG 等开源方案只是聚合第三方搜索引擎结果，不具备独立索引能力。

### 三种搜索方式对比

| 特性 | DashScope | Tavily | DDGS |
|------|-----------|--------|------|
| API Key 需求 | 需要（阿里 DashScope） | 需要 | 不需要 |
| 搜索结果质量 | 高（夸克搜索） | 高（专业搜索引擎） | 较低 |
| 费用 | 按量计费 | 有免费额度 | 免费 |
| 安装包 | `dashscope` | `tavily-python` | `ddgs` |
| LangChain 集成 | 需自定义工具 | 官方 Retriever | 需自定义工具 |

### DashScope 搜索关键代码

```python
from dashscope import Generation

response = Generation.call(
    model='qwen3-max',
    prompt=query,
    enable_search=True,  # 开启联网搜索
    result_format='message'
)
```

### Tavily 搜索关键代码

```python
from langchain_community.retrievers import TavilySearchAPIRetriever

retriever = TavilySearchAPIRetriever(k=3)
results = retriever.invoke(query)

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)
```

### DDGS 搜索关键代码

```python
from ddgs import DDGS

ddgs = DDGS()
results = list(ddgs.text(
    query=query,
    region="wt-wt",
    timelimit='y',
    safesearch='off',
    max_results=3,
))
```

### Agent 封装模式

三种搜索方式均通过 `@tool` 装饰器封装为 LangChain 工具，再通过 `create_agent()` 注入 Agent。这体现了 LangChain 的工具抽象思想——任何外部能力（搜索、计算、数据库等）都可以统一包装为 Tool，供 Agent 自主调用。

## 参考答案汇总

### 基础题

1. **三项核心任务**：网页爬取、网页索引、网页检索。网页索引需要庞大的存储和计算资源，以及复杂的算法来处理海量网页的排序和存储，只有谷歌、必应等大公司具备这样的基础设施能力。

2. **三种搜索方式**：
   - **DashScope**：阿里 MaaS 平台，需 API_KEY，结果质量高（基于夸克搜索），按量计费
   - **Tavily**：LangChain 推荐的搜索服务，需 API_KEY，有免费额度，结果质量高
   - **DDGS**：免费搜索，无需 API_KEY，但结果质量较差

3. 需要设置 `enable_search=True` 参数。

4. `k=3` 表示返回最多 3 条搜索结果，用于控制检索结果的召回数量。

5. DDGS 返回每条结果包含标题（`title`）、摘要（`body`）和链接（`href`）三个字段。它是免费使用的，无需 API Key。

### 进阶题

1. **错误处理逻辑**：需要检查 `response.status_code`，仅在 `200` 时返回正常内容，否则返回错误信息。`enable_search=True` 会触发 DashScope 内部的联网搜索机制，模型在生成回答前先检索互联网信息。

2. **质量差异**：Tavily 拥有专业的网页索引和检索能力，能返回高质量、结构化的搜索结果；DDGS 依赖 DuckDuckGo 的公共 API，搜索结果较为粗糙，缺少深度索引。项目选择时应优先考虑 Tavily 或 DashScope，DDGS 仅适合快速原型验证。

3. **`format_docs` 的作用**：将多条搜索结果的 `page_content` 合并为一个字符串，方便 LLM 一次性阅读所有上下文。合并后的内容更紧凑，减少了 Agent 推理时的 Token 消耗，同时保持了信息的完整性。

### 编程/实践题

```python
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain.agents import create_agent
from langchain.tools import tool
from langchain_community.retrievers import TavilySearchAPIRetriever

_ = load_dotenv()

llm = ChatOpenAI(
    model="qwen-max",
    temperature=0.7,
    api_key=os.getenv("DASHSCOPE_API_KEY"),
    base_url=os.getenv("DASHSCOPE_BASE_URL"),
)

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

@tool
def tavily_search(query: str) -> str:
    """使用 Tavily 搜索 API 搜索互联网信息"""
    retriever = TavilySearchAPIRetriever(k=3)
    contents = retriever.invoke(query)
    if len(contents) > 0:
        return format_docs(contents)
    else:
        return "没有搜索结果"

agent = create_agent(
    model=llm,
    tools=[tavily_search],
    system_prompt="你是一个智能助手，回答前必须使用工具搜索互联网信息",
)

response = agent.invoke({
    "messages": [{"role": "user", "content": "今天最重要的科技新闻是什么？"}]
})

for message in response['messages']:
    message.pretty_print()
```
