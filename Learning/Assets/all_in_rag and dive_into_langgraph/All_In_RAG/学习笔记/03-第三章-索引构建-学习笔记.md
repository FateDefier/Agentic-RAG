## 要点整理

### 1. 向量嵌入（Embedding）
- **定义**：将文本/图像等数据转换为低维稠密向量
- **核心原则**：语义相似的对象在向量空间中距离更近
- **度量方法**：余弦相似度、点积、欧氏距离
- **在 RAG 中的作用**：语义检索的基础，直接决定检索质量

### 2. Embedding 技术演进
| 阶段 | 代表模型 | 特点 | 局限 |
|------|----------|------|------|
| 静态词嵌入 | Word2Vec, GloVe | 上下文无关 | 无法处理一词多义 |
| 动态上下文嵌入 | BERT | 上下文相关 | 计算量大 |
| RAG 专用嵌入 | INSTRUCTOR, BGE | 领域自适应 | 需要微调 |

### 3. 嵌入模型训练
- **MLM（掩码语言模型）**：预测被遮盖的词，学习上下文关系
- **NSP（下一句预测）**：判断句子关系（RoBERTa 已移除）
- **对比学习**：拉近相似样本，推远不相似样本

### 4. 向量数据库核心
| 功能 | 说明 |
|------|------|
| 相似性搜索 | ANN（近似最近邻），毫秒级响应 |
| 高维存储 | 存储和管理高维向量 |
| 混合查询 | 向量搜索 + 标量过滤 |
| 分布式扩展 | 水平扩展，高可用 |

### 5. 索引算法
| 算法 | 原理 | 适用场景 |
|------|------|----------|
| HNSW | 分层可导航小世界图 | 高性能检索，内存充足 |
| IVF | 倒排文件索引 + 聚类 | 大规模数据，可调节 |
| PQ | 乘积量化压缩向量 | 节省存储空间 |
| LSH | 局部敏感哈希 | 近似搜索，快速索引 |

### 6. 主流向量数据库
| 产品 | 类型 | 特点 | 适用场景 |
|------|------|------|----------|
| Chroma | 轻量级 | 零配置，本地运行 | 原型开发、小规模 |
| FAISS | 算法库 | 高性能，本地存储 | 原型设计、中小型应用 |
| Milvus | 分布式 | GPU加速，亿级检索 | 大规模生产环境 |
| Pinecone | 托管服务 | Serverless，99.95%SLA | 企业级生产 |
| Qdrant | 开源 | Rust开发，高RPS | 性能敏感场景 |
| Weaviate | 开源 | GraphQL，AI集成 | 多模态处理 |

### 7. 多模态嵌入
- **CLIP**：双编码器（文本编码器 + 图像编码器），对比学习训练
- **BGE-Visualized**：支持图片和文本的统一嵌入
- **应用**：图文混合检索、以图搜文、以文搜图

## 参考答案汇总

> ⚠️ 先自己作答，再对照检查

### 基础题答案

1. Embedding 是将文本、图像等数据转换为数学向量的技术。在 RAG 中，它将知识库文档转换为向量存入向量数据库，并通过语义相似度搜索实现高质量检索。

2. ① **余弦相似度**：计算向量夹角余弦值，值越近1越相似；② **点积**：向量归一化后等价于余弦相似度；③ **欧氏距离**：空间直线距离，越小越相似。

3. ① 高效相似性搜索（ANN 毫秒级）；② 专为高维向量优化存储；③ 支持向量+标量混合查询；④ 分布式高可用。

4-6. 详见要点整理各表格。

### 进阶题参考答案

1. **自动编码器**：通过编码-解码重构学习，优点是无监督，缺点是可能学习低级特征。**对比学习**：通过正负样本对学习，优点是能学习判别性特征，缺点是需要大量负样本。**CLIP**：双编码器+对比学习，优点是图文对齐效果好，缺点是需要大规模图文对数据。

2. **HNSW**：构建多层图结构，搜索时从顶层到底层逐层查找。适合高精度、低延迟场景，但内存消耗大。
**IVF**：通过 K-means 聚类划分空间，搜索时只搜索最相近的簇。适合大规模数据，可调参数平衡速度和精度。
**PQ**：将高维向量分割编码压缩。适合存储受限、需要节省空间的场景。

3. 原型开发选 Chroma/FAISS（轻量快速）；中小规模（百万级）选 Qdrant/Weaviate；大规模生产（亿级）选 Milvus/Pinecone；需要 GPU 加速选 Milvus。

### 编程题参考实现

```python
from langchain_community.document_loaders import TextLoader
from langchain.text_splitter import CharacterTextSplitter
from langchain_community.embeddings import HuggingFaceEmbeddings
from langchain_community.vectorstores import FAISS

# 加载文档
loader = TextLoader("document.txt")
documents = loader.load()

# 分块
splitter = CharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(documents)

# 嵌入 + FAISS 索引
embeddings = HuggingFaceEmbeddings(model_name="BAAI/bge-small-zh-v1.5")
vectorstore = FAISS.from_documents(chunks, embeddings)

# 相似度搜索
query = "你的问题"
results = vectorstore.similarity_search(query, k=3)
for doc in results:
    print(doc.page_content[:200])
```

### 大厂面试题参考解答

**题目 1（字节跳动）：**
选型指标：MTEB 排行榜分数（涵盖分类、聚类、检索等任务）、模型参数量（资源消耗）、上下文窗口大小（支持的最大 token 数）、推理速度。常用模型：bge-small-zh-v1.5（轻量）、bge-large-zh-v1.5（高精度）、text-embedding-ada-002（OpenAI）、m3e（国产开源）。

**题目 2（阿里巴巴）：**
百万级：IVF_FLAT（IVF 索引，聚类数 nl=sqrt(n)），查询快，索引建得也快。
千万级：IVF_SQ8（量化压缩，减少内存），或 HNSW 图索引。
亿级：分布式部署 Milvus，分区分片，配合 PQ 量化。
策略：根据召回率要求调整 nprobe（搜索簇数）、efConstruction（HNSW 构建参数）。

**题目 3（腾讯）：**
应用场景：电商（图文搜索）、医疗（影像+报告）、内容审核（图片+文本）。实现方式：使用 CLIP/BGE-Visualized 等统一多模态嵌入模型，将图文映射到同一语义空间，执行跨模态相似度搜索。核心：需要对齐图文语义空间，确保图文相似度计算有效。
