## 要点整理

### 1. 核心优化方向
| 优化方面 | 基础版（第八章） | 优化版（第九章） |
|----------|----------------|----------------|
| **检索策略** | 单一向量检索 | 图RAG + 向量检索混合 |
| **数据结构** | 文本块索引 | 知识图谱 + 向量索引 |
| **查询处理** | 简单语义搜索 | 智能查询路由 |
| **数据库** | FAISS（本地） | Milvus（分布式） |
| **多模态支持** | 文本仅 | 文本+图像多模态 |

### 2. 图RAG 架构
- **核心思想**：利用知识图谱的图结构增强检索能力
- **数据建模**：提取实体和关系，构建三元组
- **存储方式**：图数据库（Neo4j）+ 向量数据库（Milvus）
- **检索流程**：
  1. 用户查询 → 查询路由判断类型
  2. 关系型问题 → 图遍历检索
  3. 语义型问题 → 向量相似度检索
  4. 融合结果 → LLM 生成

### 3. Milvus 索引构建
- **多模态嵌入**：文本用 bge、图像用 CLIP/ViT
- **统一索引空间**：不同模态映射到同一向量空间
- **分片策略**：按数据源/领域分片
- **索引类型**：IVF_FLAT / HNSW 根据数据量选择

### 4. 智能查询路由
| 查询类型 | 特征 | 路由目标 |
|----------|------|----------|
| **事实类** | "X 是什么？" | 知识图谱检索 |
| **关系类** | "X 和 Y 的关系？" | 图遍历检索 |
| **语义类** | "解释一下 X" | 向量语义检索 |
| **多模态** | "包含 X 的图片" | 多模态检索 |
| **混合类** | 多类型结合 | 多路融合检索 |

### 5. 系统提升
- **检索精度**：提升 15-30%（图+向量双通道）
- **多跳推理**：从仅支持单跳提升到支持 2-3 跳推理
- **响应质量**：查询路由确保走最合适的检索路径
- **可扩展性**：Milvus 支持亿级向量，水平扩展

### 基础题答案

1. 核心目标：提升检索精度和多跳推理能力。优化方面：引入图RAG、升级 Milvus 向量库、增加智能查询路由、支持多模态检索。

2. 图RAG 的核心思想是利用知识图谱的图结构（实体-关系-实体）来增强检索。相比纯文本块检索，图结构能更好地支持多跳推理和关系型问答。

3. 通过多模态嵌入模型（如 CLIP/BGE-Visualized）将文本和图像映射到同一语义空间，然后统一存入 Milvus。检索时根据用户输入的模态类型，自动选择对应编码器。

4. 智能查询路由根据用户问题的类型和特征，自动选择最合适的检索策略（图检索/向量检索/混合检索），提高检索效率和质量。

5. ① 实体识别（从文档中抽取实体）；② 关系抽取（判断实体间关系）；③ 三元组构建；④ 图数据导入 Neo4j；⑤ 建立实体到向量的映射索引。

### 进阶题参考答案

1. 完整检索流程：用户查询 → 查询路由判断 → 如果是关系型问题（如"A 和 B 的关系"）：从图数据库遍历实体和关系 → 生成子图 → 子图转文本 → 融合向量检索结果 → LLM 生成。如果是语义型问题（如"解释概念"）：走传统向量检索 → Rerank → LLM 生成。如果是混合型：两者都走 → RRF 融合 → LLM 生成。

2. 决策依据：① 问题中的关键词（如"关系""对比"暗示图检索）；② 问题中包含的实体类型；③ LLM 对问题的分类（使用 LLM 判断）；④ 历史查询模式匹配。策略选择：简单事实→图检索；深层概念→向量检索；复杂推理→混合检索。

3. ① 检索精度提升（图结构补充向量检索盲区）；② 多跳推理能力增强（图遍历支持关系链推理）；③ 查询响应更精准（路由确保走最合适路径）；④ 多模态支持扩展了应用范围；⑤ Milvus 分布式部署提升性能和扩展性。

# 智能查询路由示例
import re
from typing import Literal

def classify_query(query: str) -> Literal["graph", "vector", "hybrid"]:
    """基于规则和关键词的查询分类"""
    graph_keywords = ["关系", "联系", "关联", "路径", "链", "网络",
                      "对比", "比较", "依赖", "因果", "影响"]
    hybrid_keywords = ["整体", "全面", "综合分析", "概述"]

    # 检查是否包含图检索关键词
    for kw in graph_keywords:
        if kw in query and len(query) < 30:  # 短查询倾向于图检索
            return "graph"

    # 包含 "关系" 类词汇且涉及多实体
    entities = re.findall(r'[\u4e00-\u9fa5]{2,4}', query)
    if len(set(entities)) >= 3:
        return "hybrid"

    return "vector"

def route_query(query: str):
    query_type = classify_query(query)
    if query_type == "graph":
        # 图数据库检索
        results = neo4j_search(query)
    elif query_type == "vector":
        # 向量相似度检索
        results = vector_search(query)
    else:
        # 混合检索
        graph_results = neo4j_search(query)
        vector_results = vector_search(query)
        results = fuse_results(graph_results, vector_results)

    return generate_answer(query, results)

### 大厂面试题参考解答

**题目 1（字节跳动）：**
智能查询路由设计：① 问题分类层（LLM 或规则分类器识别问题类型）；② 策略选择层（映射到不同检索策略——向量/图/搜索引擎/混合）；③ 结果融合层（多路结果去重、重排、融合）；④ 兜底策略（默认走向量检索）。实现方式：规则引擎（关键词匹配）或 LLM 驱动的动态路由，或两者结合。

**题目 2（阿里）：**
多模态索引设计：① 不同模态用专用编码器（文本-BGE，图像-CLIP，表格-Table Transformer）；② 对齐到统一语义空间，支持跨模态检索；③ 在 Milvus 中建立多个集合（collection）或通过标签字段区分模态。检索策略：查询时自动编码用户输入到相应模态向量，同时检索向量库，可选按模态过滤。
