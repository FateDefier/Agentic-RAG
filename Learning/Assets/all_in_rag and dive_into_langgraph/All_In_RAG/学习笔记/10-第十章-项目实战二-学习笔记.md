## 要点整理

### 1. 项目实战二规划方向
根据项目 README 和 RAG 技术趋势，实战二可能涵盖：

| 方向 | 说明 |
|------|------|
| **Agentic RAG** | RAG 作为 Agent 的工具，Agent 控制检索时机 |
| **多模态 RAG** | 图像/视频/音频综合检索生成 |
| **实时 RAG** | 实时数据流检索（新闻/社交媒体） |
| **企业级 RAG** | 多租户、权限管理、审计日志 |
| **RAG + Fine-tuning** | 混合方案的最佳实践 |

### 2. 从实战一到实战二的进阶路径
- **检索端**：从单路向量检索 → 多路混合检索 + 自适应路由
- **索引端**：从单模态索引 → 多模态统一索引
- **生成端**：从简单 Prompt → 结构化输出 + 引用标注
- **评估端**：从人工评估 → 自动化评估体系（RAGAS）
- **部署端**：从本地 demo → 云原生分布式部署

### 3. 生产化 RAG 的关键考量
| 方面 | 关键问题 |
|------|----------|
| **性能** | 响应延迟 P99 < 3s, QPS 支持 |
| **可靠性** | 故障转移、数据备份、降级策略 |
| **安全性** | 数据隔离、权限控制、内容审核 |
| **可维护性** | 索引更新策略、模型版本管理、监控告警 |
| **成本** | API 调用成本控制、向量存储优化 |

### 4. Agentic RAG 简介
- **核心**：Agent 智能体 + RAG 检索工具
- **流程**：用户意图识别 → Agent 决策 → 调用 RAG 工具 → 多步推理 → 回答
- **优势**：自主判断何时检索、多次检索、调用其他工具
- **框架**：LangGraph、AutoGen、CrewAI

## 参考答案汇总

> ⚠️ 先自己作答，再对照检查

### 基础题答案

1. 根据项目规划，实战二预计关注更复杂的 RAG 应用场景，可能涉及 Agentic RAG、多模态 RAG 或企业级 RAG 部署。

2. 可能的技术难点：① 多模态数据对齐与索引；② Agent 意图识别与任务规划；③ 大规模分布式部署的性能优化；④ 实时数据流的增量索引；⑤ 多租户数据隔离与安全。

3. 部署：容器化（Docker/K8s）、环境管理、资源规划。运维：监控告警（Prometheus/Grafana）、日志收集、故障恢复、索引更新策略、版本管理。

### 进阶题参考答案

1. 建议选择 **Agentic RAG** 方向。理由：① 这是当前 RAG 发展的主要趋势；② 能充分发挥 LLM 的推理和规划能力；③ 与实战一形成进阶关系（从被动检索到主动检索）；④ 技术前沿、面试高频考点。

2. 重点扩展：① 检索策略多元化（从单路到多路路由）；② 引入外部工具（搜索引擎、API、数据库）；③ 支持多轮对话中的上下文维护；④ 增加评估与监控体系；⑤ 生产化部署（性能、可靠性、安全）。

### 大厂面试题参考解答

**题目 1（字节跳动）：**
传统 RAG 是固定流程：先检索后生成。Agentic RAG 是把 RAG 作为 Agent 工具箱中的一个工具，Agent 自主判断何时检索、是否需要多次检索、是否需要补充其他工具。实现方式：用 LangGraph 或 AutoGen 定义 Agent 行为，RAG 检索器作为 Tool。当 Agent 判定问题需要外部知识时，调用 Tool 检索 → 评估结果 → 决定是否需补充检索 → 生成最终答案。

**题目 2（腾讯）：**
关键考量：① **数据层**：文档解析质量、分块策略、索引更新频率；② **检索层**：混合检索+ Rerank + 查询路由的高可用设计；③ **生成层**：Prompt 模板管理、输出质量控制、兜底策略；④ **性能层**：缓存（KV Cache + 结果缓存）、异步处理、分布式部署；⑤ **监控层**：全链路 Trace、QA 质量评分、用户反馈采集；⑥ **运维层**：模型版本管理、AB 测试、灰度发布。
