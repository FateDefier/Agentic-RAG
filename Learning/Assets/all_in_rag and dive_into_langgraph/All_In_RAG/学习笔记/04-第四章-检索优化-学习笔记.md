## 要点整理

### 1. 混合检索
- **定义**：结合稀疏向量（关键词匹配）和密集向量（语义理解）的搜索技术
- **稀疏向量**：维度高、稀疏、基于词频（BM25），可解释性强，精确匹配
- **密集向量**：低维稠密、基于深度学习，语义理解强，泛化能力好
- **融合策略**：加权求和（RRF）、级联搜索、并行搜索后融合

### 2. BM25 算法
- **核心公式**：\[ Score(Q,D) = \sum IDF(q_i) \cdot \frac{f(q_i,D) \cdot (k_1+1)}{f(q_i,D) + k_1 \cdot (1-b + b \cdot |D|/avgdl)} \]
- **参数**：\(k_1\)（词频饱和度）、\(b\)（文档长度归一化）
- **特点**：关键词精确匹配，对专业术语效果好

### 3. 查询构建
- **目标**：提高用户查询的精度，解决口语化/模糊查询问题
- **方法**：
  - 查询重写（Rewrite）：用 LLM 将用户问题改写为更专业形式
  - 查询扩展（Expansion）：添加同义词或相关术语
  - 查询分解（Decomposition）：将复杂问题拆解为子问题

### 4. 查询重构与分发
- **策略**：
  - **假设文档嵌入（HyDE）**：先生成假设答案，再用答案检索
  - **子问题分解**：多跳推理时分解为子问题序列
  - **多路分发**：不同策略路由到不同检索器（向量/知识图谱/搜索引擎）

### 5. Text2SQL
- **原理**：将自然语言问题转换为 SQL 查询语句
- **主要挑战**：复杂嵌套查询、表结构理解、领域术语对齐
- **技术方案**：LLM 直接生成（少样本提示）、Schema Linking + SQL 生成、NL2SQL 专用模型

### 6. 检索进阶技术
| 技术 | 说明 | 优势 |
|------|------|------|
| HyDE | 假设文档嵌入 | 零样本场景检索提升 |
| Rerank | 交叉编码器重排序 | 提升 Top-K 准确率 |
| CRAG | 纠正性 RAG | 自动处理低质量检索 |
| Self-RAG | 自反思 RAG | 自适应决定是否检索 |
| Graph RAG | 知识图谱增强 | 多跳推理能力 |

## 参考答案汇总

> ⚠️ 先自己作答，再对照检查

### 基础题答案

1. 混合检索是结合稀疏向量（关键词匹配）和密集向量（语义理解）的搜索技术，旨在利用两者的互补优势。

2. **稀疏向量**：高维（≈词汇表大小）、绝大多数元素为0、每个维度对应具体词、可解释性强、精确匹配。**密集向量**：低维稠密（几百维）、元素非零、每个维度无具体含义、语义理解强。

3. BM25 基于词频（TF）和逆文档频率（IDF）对文档排序。超参数：k1（控制词频饱和度，默认1.2-2.0）、b（文档长度归一化，默认0.75）。

4. 解决用户查询口语化、信息不全、模糊等问题，通过重写/扩展/分解提高检索准确率。

5. Text2SQL 用 LLM 将自然语言转换为 SQL 查询。应用：数据库查询、数据分析、商业智能。

6. 查询重构是用 LLM 优化用户查询。策略：查询重写（规范化）、查询扩展（加同义词）、查询分解（拆子问题）。

### 进阶题参考答案

1. **稠密检索**：优点-语义理解强，泛化好；缺点-对稀有实体/专业术语匹配差，可解释性差。**稀疏检索**：优点-精确匹配强，速度快，可解释；缺点-语义鸿沟。融合原因：两者互补，稠密检索负责理解意图，稀疏检索负责精确匹配。

2. 查询路由根据用户问题的类型/领域，将其分发到不同的检索器（如向量检索处理语义问题、知识图谱处理关系问题、搜索引擎处理实时问题）。例如：查询"苹果公司2024年营收"→知识图谱检索；"Transformer 原理"→向量语义检索；"今天天气"→搜索引擎。

3. 挑战：复杂查询解析、多表 JOIN、领域特定术语、歧义消解。方案：① Schema Linking（明确涉及的表和列）；② Few-shot Prompting（提供 SQL 示例）；③ 中间表示（先转语义表示再生成 SQL）；④ 领域微调 LLM。

### 编程题参考实现

```python
# 混合检索实现（BM25 + 向量检索）
from rank_bm25 import BM25Okapi
from sentence_transformers import SentenceTransformer
import numpy as np

# 示例文档
documents = [
    "苹果公司发布了iPhone 15",
    "红富士苹果是水果的一种",
    "苹果手机搭载A16芯片"
]
tokenized_docs = [doc.split() for doc in documents]

# BM25 稀疏检索
bm25 = BM25Okapi(tokenized_docs)
query = "苹果手机"
tokenized_query = query.split()
sparse_scores = bm25.get_scores(tokenized_query)  # [0.0, 0.0, 0.88]

# 密集向量检索
model = SentenceTransformer('BAAI/bge-small-zh-v1.5')
doc_embeddings = model.encode(documents)
query_embedding = model.encode([query])
dense_scores = np.dot(doc_embeddings, query_embedding.T).flatten()

# 融合（RRF 加权）
def rrf(scores, k=60):
    rankings = np.argsort(scores)[::-1]
    return 1 / (np.arange(len(scores)) + k)

sparse_rrf = rrf(sparse_scores)
dense_rrf = rrf(dense_scores)
alpha = 0.3  # 稀疏权重
final_scores = alpha * sparse_rrf + (1-alpha) * dense_rrf

best_idx = np.argmax(final_scores)
print(f"最佳匹配: {documents[best_idx]}")
```

### 大厂面试题参考解答

**题目 1（字节跳动）：**
纯向量检索无法处理精确关键词匹配（如产品型号、人名、代码）。混合检索结合 BM25（精确匹配）+ 向量检索（语义理解），可在各种查询类型下保持稳定的检索效果。实现通常用 RRF（倒数排序融合）或加权平均融合两路结果。

**题目 2（阿里巴巴）：**
用户查询通常口语化、信息不全（如"苹果股价咋样"→"Apple Inc.最新股票价格"）。重写策略：① 查询规范化（补全信息）；② 同义词扩展（"苹果"→"Apple/苹果公司"）；③ 假设文档嵌入（HyDE：先假设答案再检索）；④ 多轮对话上下文增强。

**题目 3（腾讯）：**
Lost in the Middle 指 LLM 在处理长上下文时，更关注开头和结尾的信息，忽略中间部分。缓解措施：① 通过 Rerank 把最相关文档放在开头或结尾；② 合理控制分块大小；③ 使用 Lost-in-the-Middle 优化的 Prompt 模板；④ 考虑 CoT（思维链）引导模型遍历全部上下文。
