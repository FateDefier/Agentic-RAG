## 要点整理

### 1. 生成集成核心任务
- **上下文整合**：将检索到的文档片段和用户问题有效组合
- **指令引导**：通过 Prompt 引导 LLM 基于检索内容生成
- **格式控制**：确保输出符合预期格式（JSON/文本/表格等）
- **忠实度保障**：防止 LLM 脱离上下文编造信息

### 2. 格式化生成方法
| 方法 | 说明 | 适用场景 |
|------|------|----------|
| Prompt 约束 | 在 Prompt 中明确指定输出格式 | 通用场景 |
| Output Parser | 使用框架的输出解析器（LangChain） | 需要结构化数据 |
| JSON Mode | 调用 API 的 response_format 参数 | API 调用 |
| 约束解码 | 使用指导解码约束输出格式 | 严格要求格式 |

### 3. 确保忠实度的策略
- **Prompt 约束**：明确要求"基于以下内容回答，不要编造"
- **提供不相关答案的兜底**：允许模型回答"找不到相关信息"
- **引用标注**：要求模型标注答案来源（Citation）
- **Rerank 保障**：确保进入上下文的文档高相关性
- **知识图谱交叉验证**：对关键事实进行多源验证

### 4. Prompt 模板设计要素
| 要素 | 说明 |
|------|------|
| **角色定义** | 明确 LLM 扮演的角色（如"专业助手"） |
| **上下文输入** | 检索到的文档块 |
| **任务指令** | 具体要做什么（回答问题/总结/翻译） |
| **格式要求** | 输出格式（文本/JSON/列表） |
| **约束条件** | 禁止编造、只基于检索内容 |
| **兜底行为** | 没有答案时如何处理 |

## 参考答案汇总

### 基础题答案

1. 生成集成阶段的主要任务是将检索到的相关文档片段与用户问题整合，通过 Prompt 引导 LLM 生成准确、可信、格式正确的回答。

2. 格式化生成是指按照预定义的格式（如 JSON、Markdown、表格）控制 LLM 的输出。它在 RAG 系统中很重要，因为下游应用通常需要结构化数据（如 API 响应、前端渲染）。

3. JSON、Markdown、XML、CSV、YAML、纯文本表格。

4. Prompt Engineering 是生成集成的核心环节，它通过设计高质量的指令，引导 LLM 正确利用检索到的上下文，控制输出格式，确保回答的准确性和忠实度。

5. 输出约束是限制 LLM 输出范围的技术。常用方法：Prompt 约束、Output Parser、JSON Mode API、约束解码（如 guidance/jsonformer）。

### 进阶题参考答案

1. ① **Prompt 约束**："请仅基于以下内容回答，不要使用你自己的知识"；② **引用强制**：要求每句话标注来源（如 [来源1][段落2]）；③ **置信度阈值**：设置 Rerank 分数阈值，低于阈值的文档不提供给 LLM；④ **无答案兜底**：当检索内容不相关时，回答"找不到相关信息"。

2. ① **Prompt 兜底**：在 Prompt 中加入"如果内容中没有答案，请说'找不到相关信息'"；② **内容过滤**：在注入 LLM 前对检索结果进行预处理过滤；③ **CRAG 纠正**：当置信度低时触发补充检索；④ **否定检测**：用分类器判断检索内容是否真正回答了问题。

### 编程题参考实现

```python
from langchain.chains import RetrievalQA
from langchain.output_parsers import StructuredOutputParser, ResponseSchema
from langchain.prompts import PromptTemplate

# 定义输出结构
response_schemas = [
    ResponseSchema(name="answer", description="问题答案"),
    ResponseSchema(name="source", description="答案来源文档"),
]
output_parser = StructuredOutputParser.from_response_schemas(response_schemas)

# 构建 Prompt 模板
template = """
基于以下上下文回答问题。请只使用上下文中的信息，不要编造。

上下文：
{context}

问题：{question}

{format_instructions}
"""
prompt = PromptTemplate(
    template=template,
    input_variables=["context", "question"],
    partial_variables={"format_instructions": output_parser.get_format_instructions()}
)

# 构建 RAG 链
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=vectorstore.as_retriever(),
    chain_type_kwargs={"prompt": prompt}
)

response = qa_chain.invoke("你的问题")
print(response)  # 输出 JSON 格式
```

### 大厂面试题参考解答

**题目 1（字节跳动）：**
幻觉原因：① 检索内容不相关；② 上下文信息不足；③ LLM 过度泛化。缓解策略：① Prompt 约束强制基于上下文；② Rerank 确保高质量输入；③ 无答案兜底机制；④ 引用溯源；⑤ 定期评估 Faithfulness 指标。

**题目 2（阿里）：**
使用 Output Parser（如 PydanticOutputParser）定义答案和来源字段。在 Prompt 中明确要求："请以 JSON 格式回答，包含 answer 和 source 字段。answer 是问题答案，source 是所用文档的索引编号。"

**题目 3（腾讯）：**
Prompt 设计关键："请仔细阅读以下文档内容。如果文档中不包含问题的答案，请直接回复'找不到相关信息'，不要尝试自己回答。"同时通过 Rerank 确保高相关性，并设置 LLM 的 temperature 参数较低（0-0.3），减少创造性回答的倾向。
